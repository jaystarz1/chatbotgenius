Privacy Policy

Medical PET/CT Transcription API Service

Effective Date: August 4, 2025
Last Updated: August 4, 2025

1. Overview

This Privacy Policy describes how The Chatbot Genius Medical Transcription API ("we," "our," or "the Service") handles information when providing medical transcription services that convert doctor dictation into formatted PET/CT reports.

Important: This service is designed for healthcare professionals only and processes medical dictation for the sole purpose of generating properly formatted medical reports.

2. Information We Process

Our API processes the following types of information:

  • Medical Dictation Text: Raw medical dictation provided by healthcare professionals
  • Processing Metadata: Technical information about API requests (timestamps, processing time, request parameters)
  • Formatted Reports: The structured medical reports generated from the dictation

3. How We Handle Medical Information

3.1 No Storage Policy

We do NOT store, save, or retain any medical information processed through our API:

  • Medical dictation is processed in real-time and immediately discarded
  • Generated reports are returned to the requesting system and not retained
  • No patient data, medical records, or health information is stored on our servers
  • Processing occurs entirely in memory and is cleared after each request

3.2 Processing Purpose

Medical information is processed solely for:

  • Converting unstructured dictation into formatted PET/CT reports
  • Applying medical formatting standards and terminology corrections
  • Ensuring proper structure and mandatory phrasing requirements

4. Data Security

We implement appropriate technical safeguards:

  • Encryption in Transit: All API communications use HTTPS/TLS encryption
  • Serverless Architecture: Processing occurs in isolated, stateless functions
  • No Persistent Storage: No databases or file systems store medical data
  • Memory Clearance: All processing memory is automatically cleared after each request

5. Third-Party Integration

Our API is designed to integrate with ChatGPT Custom Actions:

  • Medical dictation may be transmitted through ChatGPT's infrastructure
  • Users should review OpenAI's privacy policy for their data handling practices
  • We do not control or have access to ChatGPT's data processing or storage

6. Compliance

While our service processes medical information, we:

  • Do not store protected health information (PHI)
  • Process data only as a technical formatting service
  • Recommend healthcare professionals ensure their own HIPAA compliance
  • Advise users to de-identify patient information when possible

7. Logging and Monitoring

We maintain minimal technical logs for service operation:

  • API request timestamps and response codes
  • Processing performance metrics
  • Error logs for debugging (without medical content)
  • These logs do not contain medical dictation or patient information

8. User Responsibilities

Healthcare professionals using this service are responsible for:

  • Ensuring compliance with applicable privacy laws (HIPAA, etc.)
  • De-identifying patient information when appropriate
  • Securing their own systems and access to the API
  • Understanding their organization's data handling policies

9. Service Limitations

This API service:

  • Is a technical formatting tool only
  • Does not provide medical advice or diagnosis
  • Should not be used as the sole basis for medical decisions
  • Requires healthcare professional review of all generated reports

10. Changes to This Policy

We may update this privacy policy to reflect changes in our practices or applicable laws. Users will be notified of material changes through the API documentation or service announcements.

11. Data Subject Rights

Since we do not store personal or medical information, there is no persistent data to access, correct, or delete. All processing is real-time and temporary.
